## 貫穿全篇的幾條原則

- **資料模型先於 UI**：使用者「看不到 X」，先問「我們記得 X 嗎」。
- **重構只服務於功能**：不獨立排期大重寫；現代化 ≠ 換框架，先界定硬限制。
- **相容性是硬需求**：對已部署的資料格式只做「新增、相容、可降級」，用 round-trip 測試釘死。
- **可信度來自可驗證的不變量**：冪等、內容雜湊、結構性測試，勝過「看起來沒問題」。
- **誠實工程**：測試綠 ≠ 功能正確；區分「已自動驗證」與「需人工驗證」；不畫需假造資料的圖。
